## Стэк

Используй NestJs + TypeScript

## Требования

- Бэкенд предоставляет API по спецификации api/generated/openapi.yaml
- API бэкенда предназначен для отдельного фронтенд-клиента.
- Основные бизнес-правила бронирования реализуются на стороне бэкенда.
- Для этого шага отдельная база данных не нужна. Достаточно простого хранилища в памяти:
  после перезапуска сервиса данные могут сбрасываться.
- Приложение запускается на порту 3000
